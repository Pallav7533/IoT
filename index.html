Water Quality Monitoring system

The aim of a water quality monitoring system is to continuously assess and maintain the safety, cleanliness, and ecological health of water resources, ensuring they meet established environmental and public health standards. It enables timely detection and response to pollutants and other harmful changes in water quality.

Why do we need Water Quality Monitoring system

Water quality monitoring systems play a vital role in maintaining the health and safety of our water resources.Water quality monitoring ensures that drinking water is safe for human consumption, helping to prevent waterborne diseases and other health risks. Monitoring systems track pollutants and other contaminants, allowing for early detection and remediation to protect aquatic ecosystems and biodiversity.These systems help ensure compliance with environmental regulations and standards, avoiding legal issues and penalties for exceeding acceptable limits.Monitoring systems provide data to manage water resources efficiently, aiding in water conservation and sustainable use.By detecting contamination early, water quality monitoring helps reduce the risk of costly environmental damage and public health crises.Water quality data can inform the public about the state of their local water sources, promoting transparency and building trust in public institutions.Data from water quality monitoring systems support scientific research and innovation in water treatment technologies and environmental management.water quality monitoring systems are crucial for safeguarding public health, protecting the environment, ensuring regulatory compliance, and supporting sustainable management of water resources. They provide a foundation for informed decision-making and community engagement in water-related issues.

Customizable Alerts and Automation

Customizable alerts and automation in a water quality monitoring system allow users to set specific thresholds for various parameters (like pH, turbidity, or contamination levels) and receive alerts when those thresholds are crossed. These alerts can be delivered via multiple channels, such as SMS, email, or app notifications, enabling quick response to potential issues. Automation can also be integrated, allowing the system to trigger specific actions, such as opening or closing valves, activating alarms, or adjusting filtration processes, based on the data.

AI-Powered Analytics

AI-powered analytics in water quality monitoring leverage machine learning algorithms to analyze large volumes of sensor data, identifying patterns, trends, and anomalies in real time. This approach can predict potential issues before they escalate, allowing for proactive responses to maintain water quality. By continuously learning from incoming data, AI models improve their accuracy over time, leading to more precise insights and decision-making.

#include <.ESP8266WiFi.h>
#include <.ESP8266HTTPClient.h>
#include <.OneWire.h>
#include <.DallasTemperature.h>

// *IMPORTANT:* Store SSID and password securely (see below)
const char* ssid = "LAPTOP-A49IGV84 0057";
const char* password = "11111111";
const char* serverAddress = "http://192.168.137.1:8080/watertemp/water"; // Replace with your servlet URL

#define ONE_WIRE_BUS D2

OneWire oneWire(ONE_WIRE_BUS);
DallasTemperature sensors(&oneWire);

WiFiClient client;

void setup() {
  Serial.begin(9600);
  sensors.begin();

  // *Load SSID and password securely (recommended):*

  // 1. Environment Variables (ESP8266 boards):
  #ifdef ARDUINO_ESP8266_NODEMCU
  ssid = getenv("WIFI_SSID");
  password = getenv("WIFI_PASSWORD");
  #endif

  // 2. Separate Configuration File (optional):
  // You'll need to implement this logic to read the credentials from a file.

  // Connect to Wi-Fi
  WiFi.begin(ssid, password);
  Serial.println("Connecting to WiFi...");
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("Connected to the WiFi network");
}

void loop() {
  // Request temperature readings
  sensors.requestTemperatures();

  // Read temperature and format it to two decimal places
  float temperature = sensors.getTempCByIndex(0);
  String temperatureString = String(temperature, 2);

  // Send temperature data to the server
  sendDataToServer(temperatureString);

  // Delay between readings
  delay(10000); // Read temperature every 10 seconds
}

void sendDataToServer(const String& temperature) {
  HTTPClient http;

  if (http.begin(client, serverAddress)) {
    http.addHeader("Content-Type", "application/x-www-form-urlencoded");  // For POST data
    String postData = "temperature=" + temperature;
    int httpResponseCode = http.POST(postData);

    if (httpResponseCode > 0) {
      Serial.printf("HTTP Response code: %d\n", httpResponseCode);
      String response = http.getString();
      Serial.println(response);
    } else {
      Serial.printf("HTTP Error: %s\n", http.errorToString(httpResponseCode).c_str());
    }

    http.end();
  } else {
    Serial.println("Unable to connect to server.");
  }
}
